# 머신 비전을 활용한 캐리어 배치 제안

사용자가 촬영한 물품 이미지를 기반으로 물체를 인식하고, 전처리·객체 검출·분할·평가를 거쳐 캐리어 배치 제안까지 연결하는 머신비전 프로젝트입니다.






>더 자세한 결과를 보려면 doc/presentation/머신 비전을 활용한 캐리어 배치 제안
---

## 프로젝트 개요

이 프로젝트는 다양한 물품이 놓인 이미지를 입력받아 물체를 인식하고,  
배치에 필요한 정보를 추출한 뒤 캐리어 적재 제안으로 연결하는 것을 목표로 합니다.

단순한 객체 검출에 그치지 않고, 다음과 같은 전체 흐름을 하나의 파이프라인으로 구성했습니다.

- 이미지 전처리
- 투시변환 및 테이블 영역 추출
- YOLO 기반 객체 검출
- SAM2 기반 정밀 분할
- GT mask 기반 성능 평가
- 배치 최적화 로직 연동

---

## 프로젝트 목표

- 다양한 물품이 놓인 환경에서 객체를 안정적으로 인식
- 촬영 각도와 배경 영향을 줄이기 위한 전처리 파이프라인 구현
- 객체 검출 및 분할 성능을 정량적으로 평가할 수 있는 구조 설계
- YOLO와 SAM2를 결합해 더 정밀한 분할 결과 확보
- 인식 결과를 실제 캐리어 배치 추천과 연결

---

# 1. 투시변환

## 목적

- 카메라 각도로 인해 왜곡된 이미지를 **top-view** 형태로 보정
- 이후 단계에서 물체의 **실제 크기 계산**이 가능하도록 기준 좌표계를 생성
- 배치 결과를 평면상에서 시각화하기 위한 기준 이미지 확보

## 입력

- **ArUco 마커가 포함된 원본 이미지**

## 처리

- ArUco 마커 검출
- 마커의 실제 크기를 기준으로 **픽셀-실측 비율 계산**
- 원근 왜곡을 제거하는 **Perspective Transform** 수행

## 출력

- **top-view 이미지**
- **픽셀-실측 비율(scale factor, cm/pixel)**

---

# 2. 짐 Segmentation

## 목적

- 이미지 내 개별 짐을 객체 단위로 분리
- 각 객체가 어떤 물건인지 식별
- 배치 시 활용할 물체 특성(예: 딱딱함 여부)을 부여

## 입력

- **top-view 이미지**

## 처리

- 객체 segmentation 수행
- 객체별 mask 생성
- 객체 분류
- 각 객체에 대해 배치 기준 feature 부여
    - 예: `0 = 유연한 물체`, `1 = 딱딱한 물체`

## 출력

- **객체 ID**
- **객체 mask 값 (numpy array)**
- **객체 class**
- **feature (0 또는 1)**

---

# 3. OBB 탐지

## 목적

- 배치 최적화 알고리즘의 입력 형태를 **직사각형(Rectangle)** 으로 변환
- 객체의 현재 회전 각도를 추정
- 추후 객체를 위쪽 방향으로 정렬하여 배치 시각화에 활용

## 입력

- **객체 ID**
- **객체 mask 값 (numpy array)**

## 처리

- 객체별 mask에서 contour 추출
- contour 기반 **OBB(Oriented Bounding Box)** 계산
- 각 객체에 대해 중심점, 크기, 회전각 계산

## 출력

- 객체별
    - **ID**
    - **x, y** : 중심 좌표
    - **w, h** : 너비와 높이
    - **angle** : 회전 각도

---

# 4. 배치 최적화

## 목적

- 객체의 크기와 방향 정보를 바탕으로 캐리어 내부에 효율적으로 배치
- 우선순위와 물체 특성을 반영한 배치 전략 적용

## 입력

- 객체 ID별
    - **x, y, w, h, angle**
- 객체 feature
- 객체 우선순위

## 처리

- 객체를 직사각형 형태로 변환하여 배치 알고리즘 입력으로 사용
- **MaxRects 알고리즘**을 이용해 빈 공간을 최대한 효율적으로 활용
- 우선순위 및 특성에 따라 배치 순서 결정

## 출력

- **최종 배치 좌표**
- **배치 결과 시각화 이미지**
- 필요 시 공간 활용률, 배치 성공 여부 등의 지표

---
